# Dart 기본
## 변수(Variables)  
Dart에서 변수를 선언하는 방법은 크게 **타입 추론**(Typeinference)을 사용하는 방식과 **명시적 타입**(Explicit Type)을 지정하는 방식 두 가지로 나뉜다.
- `var`: 변수를 선언할 때 타입을 명시하지 않아도 Dart가 초기화된 값을 보고 타입을 추론한다. 한번 타입이 정해지면 다른 타입의 값을 넣을 수 없다. 지역 변수(함수 내부)에 사용하는 것이 dart의 권장 스타일(가이드라인)이다.
- `dynamic`: Dart의 **정적 타입 검사**(Static Analysis)를 우회해 모든 타입의 최상위 객체처럼 동작한다. 모든 타입의 값이 다 들어갈 수 있으며, 중간에 다른 타입의 값으로 변경할 수도 있다.
- 명시적 타입: 클래스나 함수의 매개변수에 사용하는 것이 dart의 권장 스타일(가이드라인)이다.
   - `int`: 정수
   - `double`: 실수
   - `String`: 문자열
   - `bool`: 참 / 거짓
## Null-Safety(널 세이프티)
Dart는 안전한 코드를 위해 기본적으로 변수에 null값을 허용하지 않는다. 서버에서 데이터를 받아오거나 사용자의 입력을 기다릴 때 아직 값이 없음을 표현하기 위해 사용한다.
- Non-nullable(기본): null값을 가질 수 없다.
- Nullable: 타입 뒤에 `?`를 붙여 null값을 가질 수 있음을 명시한다.
- Null 단언 연산자(`!`): Nullable 변수지만, 현재 이 시점에서는 절대 null값이 아님을 컴파일러에게 알릴 때 사용한다.(만약 null값을 가진다면 런타임 에러가 발생한다.)
- Null 병합 연산자(`??`): 앞의 값이 null이라면 뒤의 값을 사용하라는 의미이다.
## final, const
final과 const는 둘 다 한 번 값을 할당하면 변경할 수 없는 상수를 만든다. 하지만 값이 확정되는 시점에 차이가 있다.
- `final`: 런타임(실행 시점) 상수이다. 실행되면서 값이 한 번 할당되면 끝이다.
- `const`: 컴파일 타임(빌드 시점) 상수이다. 코드를 컴파일할 때 이미 그 값을 알고 있어야 한다.
## 컬렉션(List, Map, Set)
List, Map, Set은 데이터를 모아서 관리하는 자료구조이다.
- `List`: 순서가 있는 배열로 중복을 허용한다.
- `Map`: 키(Key)와 값(value)의 쌍으로 이루어진 사전 형태이다. 키는 중복될 수 없다.
- `Set`: 순서가 없는 배열로 중복을 허용하지 않는다. 중복된 내용이 존재한 상태로 출력을 하면 중복된 내용은 한 번만 나온다.
## 비교, 조건, 반복
- 비교 연산자 `==`같음, `!=`다름, `>`, `<`, `>=`, `<=`
- 조건문
  - `if`: **불리언(`bool`)값**을 기준으로 실행 여부를 결정하며, 복잡한 논리 연산이나 범위 비교에 적합하다. 다양한 논리 연산자(`||`or, `&&`and 등)를 사용할 수 있다.
  - `switch-case`: 특정 변수가 **어떤 값이나 형태**(Pattern)인지에 따라 실행 경로를 나눈다. case문에 추가적인 조건을 추가할 때는 `when`키워드를 사용한다.
- 반복문
- `for`: 정해진 횟수만큼 반복할 때 사용하며 `for-in`형태로 컬렉션 순회에 자주 사용된다.
- `while`: 조건이 참인 동안 계속 반복된다.
- `do-while`: 코드를 최소 한번은 먼저 실행하고 나서 조건을 확인한다.
## enum(열거형)
서로 연관된 상수의 집합을 정의할 때 사용된다.
## 함수
함수는 어떤 결과를 반환할지에 따라 리턴 타입을 명시한다.
- `void`함수: 아무런 값도 반환하지 않고 작업만 수행한다.
- 그 외 사용자 정의 함수: 특정한 타입의 값을 `return`을 이용해 반환한다.
## 파라미터(매개변수)
Dart의 함수 파라미터는 크게 세 가지 형태가 있다.
1. 위치형 매개변수(Positional Parameters): 순서대로 꼭 넣어야 하는 기본 형태이다.
2. 선택적 위치형 매개변수(Optional Positional): `[]`로 감싸며, 생략 가능하다. 생략하면 null이 되므로 기본값을 주거나 nullable로 처리해야한다.
3. 이름 지정 매개변수(Named Parameters): `{}`로 감싸며, 순서 상관 없이 이름을 지정해서 넣는다.(기본적으로 선택적이며, 필수 입력으로 만들려면 `required`를 붙인다.)
## typedef
특정 자료형이나 **함수의 시그니처**(형태)에 별명을 붙여주는 기능이다. 코드가 길어지는 것을 방지하고 가독성을 높일 때 유용하다.
