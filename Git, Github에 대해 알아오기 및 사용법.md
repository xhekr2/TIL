# Git, Github에 대해 알아오기 및 사용법
## Git과 Github는 무엇인가?
1. Git   
Git은 **Linus Torvalds**가 **Linux Kernel** 개발을 위해 만든 **분산형 버전 관리 시스템**(Distributed Version Control System, DVCS)으로,
소프트웨어 개발 과정에서 **파일의 변경 이력을 효율적으로 관리하고 협업을 가능하게 하는 시스템**임.
   - **버전 관리**: 코드를 작성하다가 실수를 하거나 과거의 상태로 돌아가야 할 때, Git을 사용하면 저장해 둔 특정 시점(버전)으로 쉽게 되돌아갈 수 있음.
   - **독립적인 작업 환경** (Branch): 여러 기능을 동시에 개발해야 할 때, 원본 코드에 영향을 주지 않고 자신만의 독립적인 작업 공간(브랜치)을 만들어 안전하게 코딩할 수 있으며 작업이 완료되면 원본 코드에 병합(Merge)함.
   - **로컬 중심**: Git은 기본적으로 개발자의 개인 컴퓨터(로컬) 내에서 작동하는 소프트웨어임. 인터넷이 연결되어 있지 않아도 코드를 수정하고 버전을 기록(Commit)할 수 있음.
2. Github   
Github는 **Git을 사용하는 프로젝트를 지원하는 클라우드 기반 호스팅 서비스**임.
   - **원격 저장소** (Remote Repository): 내 컴퓨터(로컬)에 있는 Git 저장소를 인터넷(클라우드) 상에 안전하게 백업하고 저장할 수 있는 공간을 제공함.
   - **협업의 극대화**: 여러 명의 개발자가 동일한 프로젝트에 참여할 때, GitHub를 통해 각자의 코드를 하나로 합치고, 코드 리뷰를 진행하며(Pull Request), 버그나 추가할 기능을 논의(Issue)할 수 있음.
   - **오픈소스 생태계**: 전 세계의 수많은 오픈소스 프로젝트가 GitHub에 공개되어 있으며, 누구나 코드를 열람하고 프로젝트 발전에 기여할 수 있음.
3. 차이점  

| 구분 | Git | Github |
|---|---|---|
| 정의 | 버전 관리 시스템(소프트웨어) | Git 저장소 호스팅 서비스(웹) |
| 동작 환경 | 내 컴퓨터(로컬) | 인터넷(클라우드) |
| 주요 목적 | 파일의 변경 이력 추적 및 버전 관리 | 원격 저장, 협업, 코드 공유 및 리뷰 |
| 사용 방식 | 주로 명령어(CLI) | 웹 브라우저(GUI) |
## Git 사용법
1. **Git 시작 및 상태 확인**  
    - `git init`  
       - 내 컴퓨터의 일반 폴더를 Git이 관리하는 저장소로 만든다.(프로젝트를 처음 시작할 때 한 번만 입력함)
    - `git clone [Github 주소]`
       - 이미 GitHub에 있는 프로젝트를 내 컴퓨터로 그대로 복사해 올 때 사용함.
    - `git status`
       - 현재 Git의 상태(수정된 파일은 무엇인지, 스테이징 되었는지 등)를 보여줌.
2. **스테이징 영역에 추가(Add)**   
   파일을 수정했다면, 버전을 만들기 위해 변경된 파일을 스테이징 영역에 추가해야함.
    - `git add [파일명]`
       - 특정 파일만 골라서 스테이징 영역에 추가.
    - `git add .`
       - 수정된 **모든 파일**을 한 번에 스테이징 영역에 추가.
3. **버전 기록(Commit)**  
   스테이징 영역에 담긴 파일들을 확정 지어 하나의 '버전'으로 저장함.
    - `git commit -m "commit메시지 작성"`
         - `-m` 뒤에 따옴표를 쓰고, 어떤 부분을 수정했는지 나중에도 알아보기 쉽게 메모를 남김.
4. **Github에 업로드하기(Push)**  
   내 컴퓨터(로컬)에 만들어진 버전을 다른 사람들과 공유하기 위해 GitHub(원격 저장소)로 보냄.
    - `git push origin main`
         - 내 로컬 저장소에서 작업한 커밋들을 원격 저장소(origin, 일반적으로 Github 저장소 URL)의 main 브랜치로 업로드하여 반영하는 명령.
## Github 사용법
1. **저장소(Repository) 만들기**  
   가장 먼저 내 프로젝트를 담을 저장소를 만들어야 함.
   1. **New Repository**: GitHub 메인 화면 오른쪽 상단의 `+` 버튼을 눌러 `New repository`를 클릭.
   2. **설정**: 이름(Repository name)을 정하고, 공개 여부(Public/Private)를 선택.
   3. **생성**: 하단의 `Create repository`를 누르면 나만의 온라인 저장소가 생김.
2. **내 컴퓨터와 GitHub 연결하기** (The Bridge)  
다음으로 내 컴퓨터(로컬)와 방금 만든 GitHub(원격)를 연결해야 함. 저장소를 만들면 나오는 주소(URL)를 복사한 뒤 터미널(CMD)에 다음을 입력.
    - `git remote add origin [복사한 주소]`   
        - 내 로컬 폴더에 "앞으로 origin이라는 이름은 이 GitHub 주소를 가리킨다!"라고 알려주는 설정임.

    - `git remote -v`
        - 연결이 잘 되었는지 확인함. 주소가 잘 뜨면 성공임.
