# Flutter란 무엇인가? 네이티브와의 차이점
## 1. 플러터(Flutter)란 무엇인가?
**플러터**는 Google이 만든 **오픈소스 UI 소프트웨어 개발 키트**(SDK)로, **하나의 코드베이스(Dart 언어 사용)로 다양한 플랫폼에서 동작하는 앱을 빠르게 개발할 수 있는 크로스 플랫폼 프레임워크**.  
- **고성능 렌더링 엔진(Skia/Impeller)을 사용**하여 네이티브에 가까운 부드러운 화면 전환과 UI를 제공함.  
- 구글의 **Dart** 언어를 사용하여 쉽고 빠르게 개발할 수 있으며, **‘Hot Reload’(핫 리로드) 기능**을 통해 코드를 수정하면 즉시 화면에 반영되는 빠른 개발 속도가 장점임.  
- 모든 것이 위젯(Widget)으로 구성되어 있어, 텍스트, 이미지, 레이아웃 등을 일관되게 설계할 수 있음.  
## 2. 네이티브(Native)와의 차이점
플러터와 네이티브의 핵심 차이는 **렌더링 방식**과 **생산성**임.
- 플러터는 **Skia 엔진을 통해 모든 UI를 직접 그리므로 UI 일관성이 뛰어나지만** 네이티브는 **각 OS의 표준 컴포넌트(View)를 사용하여 기기별 고유한 느낌**을 줄 수 있음.
- 플러터는 **하나의 코드베이스(Dart 언어)로 iOS와 Android를 동시에 개발하여 개발 속도가 빠르고 핫 리로드(Hot Reload)를 지원**하지만 네이티브는 **각 플랫폼마다 다른 언어와 코드를 작성해야 하므로 기간이 더 소요**됨.
- 플러터는 **네이티브에 근접한 고성능**을 보여줌.(자체 엔진 포함으로 앱 용량이 다소 클 수 있음.) 하지만 네이티브는 **기기 자체 컴포넌트를 사용하여 성능과 안정성이 가장 높음.**
- 플러터는 **네이티브와 통신하기 위해 'MethodChannel'을 사용해야 할 때가 있지만** 네이티브는 **최신 기능(블루투스, 센서 등)을 즉시 지원**함.

| 구분 | 플러터 | 네이티브 |
| --- | --- | --- |
| 개발 언어 | Dart | Kotlin, Swift 등 |
| 개발 생산 | 하나의 코드베이스(Dart 언어)로 iOS와 Android를 동시에 개발, 핫 리로드 지원 | 각 플랫폼마다 다른 언어와 코드를 작성 |
| 렌더링 및 UI | Skia 엔진을 통해 모든 UI를 직접 그림 | 각 OS의 표준 컴포넌트(View)를 사용 |
| 성능 및 용량 | 네이티브에 근접한 고성능 | 성능과 안정성이 가장 높음 |
| API 지원 | 네이티브와 통신하기 위해 'MethodChannel'을 사용해야할 때가 있음 | 최신 기능(블루투스, 센서 등)을 즉시 지원 |
