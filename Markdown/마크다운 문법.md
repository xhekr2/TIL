# 마크다운
마크다운의 기본 문법을 정리한다.   
마크다운은 HTML로 변환된 후 브라우저에 출력된다. 마크다운은 모든 HTML태그를 대체하지 않으며 일부 핵심 HTML으로만 변환된다.

## 제목
`<h1>`, `<h2>`, `<h3>`, `<h4>`, `<h5>`, `<h6>` 태그로 변환되는 '제목(Header)'을 표현한다
>하나의 마크다운 파일에는 하나의 `<h1>`(대제목)을 사용하는 것이 권장된다.
```markdown
# 제목1
## 제목2
### 제목3
#### 제목4
##### 제목5
###### 제목6
```

`<h1>`,`<h2>`는 아래와 같이 표현할 수도 있다.
```markdown
제목1
======

제목2
------
```

## 강조
각각 `<em>`(기울임),`<strong>`(두꺼움),`<del>`(취소선)태그로 변환되는 '강조(Emphasis)'를 표현한다.
>'*이텔릭체*와 **두껍게**를' 과 같이 한글에서는 강조 표현 다음에 띄어쓰기가 필요할 수 있다.
```markdown
이텔릭체는 *별 기호(Asterisks)* 혹은 _언더바 기호(Underscore)_ 를 사용한다.
두껍게는 **별 기호(Asterisks)** 혹은 __언더바 기호(Underscore)__ 를 2번씩 사용한다.
__*이텔릭체*와 두껍게__ 를 혼용할 수도 있다.

취소선은 ~~물결 기호(tilde)~~를 사용한다.
<u>밑줄</u>은 지원하지 않기에 직접 `<u></u>` 태그를 사용한다.
```

## 목록
`<ol>`, `<ul>`, `<li>` 태그로 변환되는 '목록(List)'을 표현한다.  
`1. `로 시작하는 항목을 작성하면 `<ol>`로 변환되며, `-`로 시작하는 항목을 작성하면 `<ul>`로 변환된다.  
들여쓰기를 통해 하위 목록을 작성할 수 있다.
>하위 목록을 작성할 때, 들여쓰기(Indent)가 띄어쓰기 2칸이 아닌 3칸 이상일 수 있다. 또한, 순서가 있거나 없는 목록을 혼합하여 사용할 수 있지만, 플랫폼에 따라 다르게 동작할 수 있다.

## 체크박스
`<input type="checkbox">` 태그로 변환되는 '체크박스(Checkbox)'를 표현한다.
목록 문법에서 대괄호(`[]`)안에 공백이나 `x`를 입력해, 할 일 목록(task list)을 만들 수 있다.
>체크박스는 일부 플랫폼에서 지원하지 않을 수 있다. 또한 대부분 플랫폼에서 직접 HTML로 변환하지 않고 플랫폼에 따라 다르게 구현된다.

## 링크
`<a>` 로 변환되는 '링크(Links)'를 표현한다.
>링크는 현재 탭에서 열리는 것이 기본적이다. 새 탭으로 열기 위해서는, 마크다운 문법으로는 `target="_blank"` 속성을 설정할 수 없기 때문에  HTML `<a>`태그를 직접 사용해야한다.

1. 링크 문법 구조
```markdown
[이름](링크)
[이름](링크 "설명")
[이름][참조]

[참조]: 링크
[참조]: 링크 "설명"
```
## 이미지
`<img>` 로 변환되는 '이미지(Images)'를 표현한다.  
링크와 비슷하지만, 앞에 `!`를 추가해야한다.

1. 이미지 문법 구조
```markdown
![대체텍스트](이미지주소)
![대체텍스트](이미지주소 "설명")
![대체텍스트][참조]

[참조]: 이미지주소
[참조]: 이미지주소 "설명"
```

2. 이미지에 링크 추가
마크다운 이미지 문법 코드를 링크 문법 코드로 감싸준다.
```markdown
[![대체텍스트](이미지주소)](링크)
```

## 코드 강조
`<pre>`,`<code>` 태그로 변환되는 '코드(Code)'를 표현한다.  
<code>`</code>백틱 기호를 사용한다.
1. 인라인  
   강조할 코드를 <code>`</code>기호로 감싸 '인라인(InLine)'코드를 표현한다.
2. 블록  
   <code>`</code>를 3번 이상 입력하고 언어(코드)를 명시해, 코드 '블록(Block)'를 표현한다.
3. 백틱 기호 사용  
   마크다운에서 백틱(<code>`</code>)기호는 코드 강조를 표현할 때 사용하는 문법 기능을 가지므로, 백틱 기호 자체를 출력하려면 기호에 이스케이프(Escape)처리가 필요하다.

   다음과 같이 `\`기호와 함께 작성하면, 백틱 기호를 출력할 수 있다. 또는 <code>`</code>에서 인라인 코드 강조를 하기 위해, \<code>태그를 활용할 수도 있다.

## 표
   `<table>` 태그로 변환되는 '표(Table)'를 표현한다.
1. 테이블 헤더 구분  
   테이블 헤더를 구분하기 위해, 일반적으로 3개 이상의 `-`(hypen/dash) 기호를 사용한다.
   테이블 헤더를 구분하며 `:`(Colons) 기호를 추가해 셀(열/칸) 안에 내용을 정리할 수 있다.
   - `---`, `:---`: 좌측 정렬
   - `:---:`: 가운데 정렬
   - `---:`: 우측 정렬

2. 표 문법 구조
```markdown
| 헤더 | 헤더 | 헤더 |
| --- | --- | --- |
| 셀 | 셀 | 셀 |
| 셀 | 셀 | 셀 |
```
>가장 좌측과 가장 우측에 있는 `|`(vertical bar) 기호는 생략 가능하다.(플랫폼에 따라 생략 불가능한 경우도 있다.)
3. 바티컬바 기호 사용  
   마크다운에서 바티컬바(`|`) 기호는 테이블을 표현할 때 사용하는 문법 기능을 가진다. 만약 테이블 안에서 바티컬바 기호 자체를 출력하려면, `\`기호와 함께 작성하는 이스케이프(Escape) 처리가 필요하다.

## 인용문
`<blockquote>` 태그로 변환되는 '인용문(BlockQuote)'을 표현한다.
1. 인용문 문법 구조
```markdown
>인용문 작성
>>중첩된 인용문 작성
>>>중중첩된 인용문 작성
```

## 원시 HTML
마크다운 문법 대신, HTML을 직접 사용(Raw HTML)할 수 있다. 마크다운 문법에서 지원하지 않는 기능을 사용할 때 유용하다.
>일부 마크다운 엔진에서는 HTML사용이 제한될 수 있다.

## 수평선
`---`, `___`, `***` 각 기호를 3개 이상 입력해, `<hr>` 태그로 변환되는 '수평선(Horizontal Rule)'을 표현한다.

## 줄바꿈
줄바꿈(Line Breaks)을 위해서는 문장 마지막에 `<br>` 태그를 직접 입력하거나, 문장 마지막에서 띄어쓰기를 2번 이상 입력한다.
>플랫폼 자체에서 자동 줄바꿈을 지원하는 경우, 이러한 방식이 필요하지 않을 수 있다. 자동 줄바꿈을 지원하지 않는 경우 `Enter(엔터)` 한 번은 같은 문단으로 인식한다.

## 주석
`<!-- -->`, `[//]: #` 기호를 사용해, 주석(Comment)을 표현한다.
